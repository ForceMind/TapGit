# 码迹（TapGit）术语映射表

## 1. 默认术语策略
- 新手模式：只显示用户语言，不显示 Git 原词
- 高级模式：可在“技术详情”区域显示 Git 对应项
- 错误提示：先用人话解释，再给可执行操作

## 2. 术语映射（Git -> 用户语言）
| Git 术语 | 默认用户文案 | 使用场景 | 备注 |
|---|---|---|---|
| repository | 项目版本库 / 版本保护空间 | 项目设置 | 默认不强调“库”概念 |
| git init | 开启版本保护 | 首次进入项目 | 按钮文案固定为“开启版本保护” |
| commit | 保存进度 | 当前修改页 | 主按钮文案：`保存进度` |
| commit message | 进度说明 | 保存弹层 | 输入框占位：`这次主要改了什么？` |
| history / log | 保存记录 / 历史时间线 | 保存记录页 | “记录”优先于“日志” |
| checkout (commit) | 恢复到这里 | 历史记录详情 | 危险操作需二次确认 |
| branch | 方案 / 新方案 | 我的方案页 | 主线可显示“主线方案” |
| switch branch | 切换方案 | 我的方案页 | 切换前提示未保存修改 |
| merge | 合并方案 | 我的方案页 | 默认目标为主线 |
| merge conflict | 修改碰到一起 | 合并流程 | 不显示“conflict”原词 |
| conflicted file | 待决定文件 | 冲突处理器 | 引导用户逐个处理 |
| push | 上传到云端 | 同步入口 | MVP 可先做占位入口 |
| pull / fetch | 获取云端最新内容 | 同步入口 | 与“上传”并列 |
| remote | 云端连接 | 设置 / 同步页 | 例如连接 GitHub |
| stash | 临时收起修改（高级） | 高级模式 | MVP 不开放主流程 |
| rebase | 整理记录（高级） | 高级模式 | MVP 不开放主流程 |
| cherry-pick | 挑选记录（高级） | 高级模式 | MVP 不开放主流程 |
| detached HEAD | 特殊状态 | 异常提示 | 给恢复指引，不抛术语 |
| ahead/behind | 有 X 条本地记录未上传 / 云端有新内容可获取 | 同步状态 | 用“记录数量”表达 |
| working tree clean | 当前没有未保存修改 | 当前修改页 | 首页也可复用 |

## 3. 标准错误文案模板
- 保存失败：`这次保存没有成功。别担心，你的当前修改还在。请重试，或查看日志。`
- 恢复失败：`恢复暂时没有完成，项目内容已保持原样。你可以重试或先创建安全快照。`
- 合并冲突：`两边改到了同一部分，需要你决定保留哪一版内容。`
- 环境缺失：`未检测到版本保护引擎（Git）。安装后即可开启版本保护。`

## 4. 危险操作确认模板
- 恢复历史确认：
`你将把项目恢复到这个时间点。当前未保存修改可能被覆盖。`
`我们会先自动创建一个安全快照，方便你回到现在。`

- 合并前确认：
`你将把“方案 B”合并到“主线”。`
`如果两边改到同一部分，我们会引导你逐项决定保留内容。`
