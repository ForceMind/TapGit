# 码迹界面统筹与 Git 场景处理方案

## 当前问题

1. 页面高度冲突：应用壳顶栏、引导条、页面标题和内容卡片同时占用垂直空间，普通窗口下必须上下滚动，核心操作不够聚焦。
2. 页面风格不一致：首页、变更、历史、备份、设置已经进入新视觉，但“试新想法”仍保留旧结构，并且部分中文文案乱码。
3. 功能入口分散：保存、上传、获取、切换方案、解决冲突、备份恢复分别藏在不同页，用户遇到问题时不知道该去哪里。
4. Git 概念泄露风险：底层动作很多，但界面应继续使用“保存、同步、试想法、恢复、整理问题”等用户语言。

## 界面原则

1. 单窗口优先：每个主页面默认适配 1366x768，不把主要操作放到首屏以下。
2. 一个页面只解决一个核心任务：变更页负责“看改动并保存”，历史页负责“找旧版本”，备份页负责“安全恢复”，试想法页负责“副本实验和带回”。
3. 顶栏降级：移除重复的大顶栏，把项目名称、当前状态放在页面自身 header 或首页项目卡片里。
4. 操作前置但不堆叠：每页最多保留 2 个主按钮，其余放入“更多处理”或设置页。
5. 危险操作必须可恢复：丢弃变更、恢复历史、合并方案前默认创建安全备份。

## 导航保留与调整

保留：
- 我的项目：打开本地项目、从 GitHub 获取、文件浏览、快速操作。
- 变更：查看当前修改、保存进度、删除文件、停止追踪、丢弃变更。
- 提交历史：查看保存记录、恢复到旧保存点。
- 备份与恢复：手动备份、自动安全备份、恢复备份。
- 试新想法：创建独立副本、切换副本、把副本带回稳定版本、处理两边改到一起。
- 设置：语言、GitHub 账号、同步、安全、日志。

去除或默认隐藏：
- 重复的顶部快捷按钮。
- 首页中重复的“下一步”大面积引导。
- 默认界面的 branch / merge / stash / reset / upstream 等术语。

新增：
- 每个页面的“问题处理提示”：当操作被未保存修改、云端账号、冲突阻塞时，给出明确下一步。
- 试新想法页的冲突处理向导：按文件逐个决定“保留稳定版本 / 保留新想法 / 手动整理”。
- Git 场景处理矩阵文档，指导后续继续补功能。

## Git 场景处理矩阵

| 底层场景 | 用户看到的说法 | 默认入口 | 处理策略 |
| --- | --- | --- | --- |
| 工作区有修改 | 有未保存的改动 | 变更 | 查看差异、保存、只保存选中文件、丢弃前自动备份 |
| 新文件 | 新增文件 | 变更 | 直接显示并允许保存；不要求理解 staged/untracked |
| 删除文件 | 删除文件 | 变更 | 引导在文件夹里删除；变更页显示删除项并可保存 |
| 不想再管理某文件 | 停止追踪 | 变更 | `rm --cached` + 写入 exclude，保留本地文件 |
| commit | 保存进度 | 变更 | 默认保存全部，支持选中文件 |
| log | 提交历史 / 保存记录 | 历史 | 时间线 + 详情 + 恢复 |
| reset --hard | 恢复到这里 | 历史/备份 | 恢复前自动安全备份 |
| branch | 试新想法 / 副本 | 试新想法 | 创建、切换、展示当前副本 |
| checkout blocked | 还有未保存修改，不能切换 | 试新想法 -> 变更 | 提示先保存或丢弃，给按钮跳转 |
| merge | 带回稳定版本 | 试新想法 | 合并成功则提示保存结果 |
| conflict | 两边改到了同一部分 | 试新想法 | 三栏对比：稳定版本 / 新想法 / 手动整理 |
| pull | 获取云端最新内容 | 设置/首页快捷 | 先检查未保存修改，失败时跳转变更页 |
| push | 上传到云端 | 设置/首页快捷 | 需要远端和账号，失败时提供账号/地址检查 |
| auth failed | 云端账号需要重新登录 | 设置 | GitHub 登录、选择账号、退出账号 |
| divergent history | 本地和云端都有新内容 | 问题处理 | MVP 先提示保存备份并手动处理；后续做可视化合并 |
| stash | 临时收起修改 | 延后 | 普通用户默认不用该词；后续可叫“暂时放一边” |
| rebase/cherry-pick | 整理记录/挑选保存点 | 高级模式 | MVP 不进入主流程 |

## 下一步落地顺序

1. 移除重复顶栏和引导条，只保留页面级 header。
2. 页面容器使用固定可用高度，内部局部滚动，避免整个应用上下滑动。
3. 重做“试新想法”页面，修复乱码并匹配新视觉。
4. 在试新想法和设置页补阻塞状态提示，引导用户去变更页保存/清理。
5. 测试所有关键页面可渲染，构建并打包。
