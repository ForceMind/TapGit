# 码迹（TapGit）产品总览

## 1. 产品一句话
码迹是一个给普通人使用的“代码进度保存与恢复”桌面应用。

它不要求用户理解 Git，也不把工程师术语直接塞进界面。用户只需要理解四件事：
- 打开或获取一个项目
- 看看现在改了什么
- 把当前可用版本保存下来
- 想试新点子时，先在安全试验区里试

## 2. 当前版本需要纠正的问题
现有版本已经能完成基础保存、恢复、试验和同步，但产品表达仍然不够对普通用户友好，主要问题是：
- 首屏入口不够清楚，没有把“打开本地项目 / 从 GitHub 获取项目”放成第一层选择
- 顶部菜单还是传统桌面软件的 `文件 / 编辑 / 查看 / 窗口 / 帮助` 思路，重复系统功能，缺少真正有用的产品动作
- 首页文案偏多，中文模式下信息密度过高，重要动作不够突出
- “试新想法”虽然已经比“我的方案”更容易懂，但还需要进一步解释它和“当前稳定版本”的关系

这次改版的目标不是再加按钮，而是重排产品结构，让第一次打开应用的人也能看懂下一步。

## 3. 产品核心原则
- 首屏先做选择，不先上工具页
- 一次只强调一个主动作，其他动作降为状态说明或次要入口
- 任何地方都优先说“用户要完成什么”，不说“底层对象是什么”
- “试新想法”必须被解释成安全试验，不是技术分支管理
- 顶部菜单只放高频、明确、可执行的动作，不放重复系统能力

## 4. 第一阶段 MVP 范围
本轮 MVP 保留并强化这些能力：
- 打开本地项目
- 从 GitHub 获取项目到本地
- 为项目开启版本保护
- 查看当前修改和变化详情
- 保存进度
- 查看历史记录并恢复到某个保存点
- 创建安全试验副本并切换
- 把试验结果带回稳定版本
- 基础云端连接和上传 / 获取

仍然延后这些能力：
- rebase
- stash
- cherry-pick
- force push
- 多远程复杂配置
- tag 管理
- 复杂 Git 图谱

## 5. 首屏产品定义
首页不是概览页，而是“开始中心”。

在没有打开项目时，首页只做三件事：
- 显示两个主入口：`打开本地项目`、`从 GitHub 获取项目`
- 显示最近项目
- 用很短的话告诉用户码迹能帮什么

在已经打开项目后，首页改成“下一步引导页”：
- 当前项目状态
- 现在先做这个
- 解锁顺序提示
- 最近项目切换

首页不再承担大量解释性文字，也不再把多个同级动作同时做成主按钮。

## 6. 顶部菜单定义
顶部菜单采用任务型结构，而不是传统开发工具结构。

建议一级菜单：
- `项目`
  - 打开本地项目
  - 从 GitHub 获取项目
  - 打开最近项目
  - 当前项目位置
- `进度`
  - 查看当前修改
  - 保存全部修改
  - 查看保存记录
- `试新想法`
  - 新建想法副本
  - 切换到稳定版本
  - 打开试新想法页
- `云端`
  - 连接 GitHub
  - 上传到云端
  - 获取最新内容
- `帮助`
  - 问题诊断
  - 导出日志
  - 关于码迹

不再把 `编辑 / 查看 / 窗口` 这些系统能力作为产品主菜单展示。保留系统默认快捷键时，以上下文输入控件自身能力为主，不额外占一级菜单。

## 7. “试新想法”功能定义
“试新想法”不是给专业 Git 用户的“分支管理”。

它在产品里的定位是：
- 当前版本已经能用
- 你想试一个有风险的新点子
- 又不想直接破坏当前稳定版本

因此它的产品表达应始终围绕三句话：
- 先从当前稳定版本开一个想法副本
- 在副本里先试
- 觉得可以，再带回稳定版本

如果用户还没有保存过一个稳定版本，就不应该鼓励他先去试新想法。

## 8. 关键用户流程
流程 A：从本地开始
打开本地项目 -> 检测是否已开启版本保护 -> 未开启则一键开启 -> 查看当前修改 -> 保存第一次稳定版本

流程 B：从 GitHub 开始
点击“从 GitHub 获取项目” -> 登录 GitHub（如需要）-> 填写仓库地址 -> 选择保存位置 -> 拉取完成并自动打开 -> 查看当前状态

流程 C：日常保存
打开项目 -> 看当前修改 -> 写一句说明 -> 保存进度 -> 在历史中看到这次保存

流程 D：回到之前
打开保存记录 -> 选中一个时间点 -> 查看影响 -> 确认恢复 -> 自动创建安全备份 -> 恢复成功

流程 E：试新点子
当前版本可用 -> 开一个想法副本 -> 在副本里修改和保存 -> 决定是否带回稳定版本

## 9. 本轮改版的验收标准
- 首屏在 5 秒内能看懂两个入口分别是什么
- 顶部菜单里的每个一级项都对应真实用户任务
- 中文首页在常见 1366 宽度下不出现大段拥挤文案
- 用户能理解“试新想法”是在安全副本里试，不是直接改当前稳定版本
- 用户能从首页完成“打开本地项目”或“从 GitHub 获取项目”两条起步流程
