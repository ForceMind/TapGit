# 码迹（TapGit）产品总览

## 1. 产品简介
码迹是一个面向普通人的 Windows 桌面“代码进度保存与恢复软件”。  
它把底层 Git 能力封装成用户可理解的任务：`保存进度`、`查看历史`、`尝试新方案`、`恢复到某个可用状态`。  
默认界面不要求用户理解 Git 术语，不把命令行概念直接按钮化。

## 2. 目标用户与核心痛点
目标用户：
- 使用 AI 编程但不懂 Git 的用户
- 独立开发者、学生、初学者
- 产品、设计、运营等会接触项目文件的人
- 需要“敢试、可退、可协作”的 Windows 用户

核心痛点：
- 不敢改代码，怕改坏后回不去
- 不知道“现在改了什么”
- 想保留阶段成果但不会用 Git
- 想并行试方案但不会分支操作
- 看见工程术语会焦虑，容易误操作

## 3. MVP 范围（第一阶段到第四阶段落地目标）
MVP 内（必须）：
- 打开本地项目并检测是否已启用版本保护
- 一键开启版本保护（底层 `git init`，前台不暴露）
- 查看当前修改列表与具体变化
- 保存进度（支持全选保存，部分文件保存作为增强能力）
- 查看历史时间线与历史详情
- 恢复到历史保存点（含确认和恢复路径）
- 创建/切换新方案
- 合并方案回主线，冲突时提供可视化选择界面
- 基础设置（Git 环境检测、日志导出、高级模式开关）

MVP 外（明确延后）：
- rebase、stash、cherry-pick、submodule、tag 管理、bisect
- force push 直出、多远程复杂策略、hooks 管理、LFS 深度支持

## 4. 产品原则（落地约束）
- 人话优先：默认界面禁用工程黑话
- 任务优先：围绕“我要做什么”，不是“Git 对象状态”
- 安全优先：危险操作必须二次确认与可恢复路径
- 默认隐藏复杂度：先保证敢用，再逐步开放高级能力

## 5. 页面结构（面向 MVP）
主框架：
- 首页（最近项目、当前状态、快速操作）
- 当前修改（修改列表、变化详情、保存进度）
- 保存记录（时间线、详情、恢复）
- 我的方案（创建、切换、合并）
- 设置（环境、文案偏好、高级模式、日志）

全局区域：
- 顶栏：当前项目、当前方案、同步状态
- 左侧导航：页面切换
- 右侧主区：任务内容
- 底部状态栏：最近操作结果与风险提示

## 6. 核心用户流程（MVP）
流程 A：首次使用  
选择项目文件夹 -> 检测版本保护 -> 一键开启版本保护 -> 进入“当前修改”

流程 B：日常保存  
查看改动 -> 输入这次说明 -> 保存进度 -> 在保存记录中看到新条目

流程 C：回退恢复  
打开保存记录 -> 预览某个历史点 -> 点击“恢复到这里” -> 风险确认 -> 自动创建安全快照 -> 恢复成功

流程 D：试新方案  
在“我的方案”创建新方案 -> 切换到新方案改动并保存 -> 对比主线 -> 合并回主线

流程 E：合并冲突处理  
合并检测到“同一处被两边修改” -> 进入冲突处理器 -> 选择保留左/右/手动编辑 -> 保存解决结果 -> 完成合并

## 7. 成功指标（MVP 验收导向）
- 新用户 10 分钟内可独立完成首次“保存进度”
- 用户能在不理解 Git 术语的前提下完成“恢复到历史点”
- 用户能完成“创建新方案 -> 切换 -> 合并回主线”
- 关键错误场景均有可执行下一步，不出现“无动作可做”的报错文案
