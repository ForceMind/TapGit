# 码迹（TapGit）信息架构

## 1. IA 总览（MVP）
```text
应用壳
├─ 首页
│  ├─ 最近项目
│  ├─ 当前项目状态卡片
│  └─ 快速操作
├─ 当前修改
│  ├─ 修改列表
│  ├─ 变化详情
│  └─ 保存进度面板
├─ 保存记录
│  ├─ 时间线
│  ├─ 记录详情
│  └─ 恢复确认弹窗
├─ 我的方案
│  ├─ 方案列表
│  ├─ 创建新方案
│  ├─ 切换方案
│  └─ 合并回主线
└─ 设置
   ├─ 环境检测
   ├─ 高级模式
   ├─ 默认保存说明建议
   └─ 日志导出
```

## 2. 关键对象模型（面向 UI，而非 Git 对象）
- `Project`：项目路径、显示名、是否开启版本保护、最近打开时间
- `CurrentState`：当前方案名、未保存文件数、最后一次保存时间
- `ChangeItem`：文件路径、变化类型（新增/修改/删除/重命名）、变更行统计
- `SaveRecord`：记录 ID、说明、时间、变更文件数、作者（本地名）
- `Plan`：方案 ID、方案名称、创建时间、最近保存摘要、是否主线
- `MergeTask`：来源方案、目标方案、状态、冲突文件列表、解决进度
- `RiskAction`：危险操作类型、确认文本、自动快照 ID、可撤销窗口状态

## 3. 页面级结构
## 3.1 首页
- 最近项目列表
- 当前项目状态卡片
- 快速按钮：`继续处理当前修改`、`保存进度`、`查看保存记录`、`创建新方案`
- 风险提醒区：例如“你有未保存内容”

## 3.2 当前修改页
- 左侧：修改文件列表（支持筛选：全部/新增/修改/删除）
- 右侧：文件变化详情（分块查看、上下文切换）
- 底部：保存说明输入框 + `保存进度`按钮
- 保护机制：有未填写说明时给出友好建议，不强制长文本

## 3.3 保存记录页
- 时间线列表（倒序）
- 详情侧栏：说明、时间、变更摘要、受影响文件
- 操作：`恢复到这里`、`查看当时变化`
- 危险操作确认：恢复前自动创建安全快照（默认开启）

## 3.4 我的方案页
- 方案列表（主线置顶）
- 操作：`创建新方案`、`切换到此方案`、`重命名方案`（MVP 可选）
- 合并入口：选择来源方案 -> 选择目标（默认主线） -> 执行合并
- 冲突处理进度条：已处理/总冲突文件

## 3.5 设置页
- Git 环境检测与修复引导
- 高级模式开关（关闭时隐藏 Git 术语）
- 日志导出（用于反馈问题）
- 关于与版本信息

## 4. 核心流程编排
## 4.1 打开项目
选择文件夹 -> 检测版本保护 -> 未启用则显示“开启版本保护” -> 完成后拉取初始状态

## 4.2 保存进度
读取修改列表 -> 用户输入说明 -> 执行保存 -> 刷新时间线与当前状态 -> Toast 成功提示

## 4.3 恢复历史
选中保存点 -> 展示影响说明 -> 二次确认 -> 自动创建安全快照 -> 执行恢复 -> 提示可撤销入口

## 4.4 新方案与合并
创建方案 -> 切换方案 -> 开发并保存 -> 发起合并 -> 处理冲突 -> 完成并生成合并记录

## 5. 状态与反馈策略
- 空状态：给出明确下一步按钮，不只显示“暂无数据”
- 加载状态：骨架屏 + 当前阶段描述（如“正在读取修改内容”）
- 错误状态：说明问题 + 可执行动作（重试/打开日志/查看帮助）
- 成功状态：轻提示 + 可继续操作建议
- 危险状态：红色确认弹窗 + 影响说明 + 恢复路径提示
