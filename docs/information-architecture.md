## 3. 关键对象模型
- `ProjectSource`：`local` / `github`
- `ProjectSummary`：项目路径、显示名、是否开启版本保护、当前所在副本、未保存修改数
- `RecentProject`：显示名、路径、最近打开时间
- `CloneDraft`：仓库地址、目标父目录、建议文件夹名
- `CurrentChange`：文件路径、变化类型、增删统计、变化详情
- `SavedRecord`：记录 ID、说明、时间、影响文件数
- `SafetyBackup`：自动备份 ID、创建时间、创建原因
- `IdeaCopy`：副本名称、最近保存说明、是否当前、是否稳定版本
- `CloudStatus`：是否已连接、待上传数量、待获取数量、当前账号状态

## 4. 首页结构

### 4.1 未打开项目
- 顶部欢迎区
- 两个主入口卡片
  - 打开本地项目
  - 从 GitHub 获取项目
- 最近项目
- 简短说明：码迹能做什么

### 4.2 已打开项目
- 当前项目概览
- 唯一主动作卡片
- 已解锁 / 未解锁步骤列表
- 最近项目

首页不再同时摆出多个同级主按钮。

## 5. 从 GitHub 获取项目流程结构
入口形式：全局弹层或向导，不单独占一个永久页面。

向导包含：
- 仓库地址输入
- GitHub 登录状态
- 目标保存位置
- 开始获取
- 成功后自动打开项目

必要时可提供两个辅助动作：
- 打开 GitHub 登录页
- 打开创建仓库页

## 6. 当前修改页结构
- 左列：改动文件列表
- 右列：文件变化详情
- 底部操作区：
  - 说明输入框
  - 保存全部
  - 只保存勾选内容

页面引导顺序固定为：
1. 先看一个文件
2. 再决定保存范围
3. 最后写说明并保存

## 7. 保存记录页结构
- 左侧：时间线
- 右侧：详情
- 下方或侧栏：安全备份
- 危险动作必须弹确认层

## 8. 试新想法页结构
- 顶部解释区：说明这不是直接改当前稳定版本
- 稳定版本卡片
- 想法副本列表
- 新建想法副本输入区
- 带回稳定版本区
- 碰撞处理区

## 9. 状态策略
- 空状态：始终带下一步动作
- 未解锁状态：明确写出“还差哪一步”
- 加载状态：只说明当前正在做什么
- 成功状态：说明结果，并给出下一步建议
- 危险状态：解释影响 + 说明恢复路径

## 10. 中文布局约束
- 首页首屏不超过两层卡片
- 主卡片内正文控制在 1 到 2 句
- 同一屏幕不重复出现相同按钮文案
- 长句优先拆成标题 + 短描述，不用整段说明堆在一张卡里
