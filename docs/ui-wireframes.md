## 4. 从 GitHub 获取项目弹层
```text
标题：从 GitHub 获取项目

仓库地址
[ https://github.com/owner/repo.git         ]

保存到哪里
[ E:\Projects                               ] [选择位置]

登录状态
GitHub：已登录 ForceMind / 尚未登录
[登录 GitHub]

[取消] [开始获取]
```

说明：
- 默认只做最短路径，不展示复杂 clone 选项
- 获取完成后自动打开项目
- 如果目标目录已存在且非空，要明确提示并阻止覆盖

## 8. 顶部菜单使用原则
- 每个一级菜单都要对应真实任务
- 不放“查看缩放”“窗口排列”这类低价值项
- 不把同一个动作在菜单、首页、顶栏里重复堆满
- 菜单适合放“全局动作”和“跨页快捷入口”

## 9. 中文视觉约束
- 中文标题优先两行内解决
- 卡片描述控制在 18 到 30 个汉字左右
- 一个区域只保留一个主按钮
- 当中文文案较长时，优先缩短句子，不靠缩小字号硬塞
