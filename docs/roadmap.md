## 2. 风险清单与规避策略
| 风险 | 影响 | 规避策略 | 降级方案 |
|---|---|---|---|
| 用户机器未安装 Git | 无法启用版本保护 | 首次启动即检测并给安装引导 | 仅允许浏览项目文件，不开放版本功能 |
| 大仓库 diff 性能差 | 页面卡顿、体验差 | 懒加载、分页、单文件按需加载 | 默认只展示摘要，全文差异手动展开 |
| 合并冲突理解成本高 | 用户恐慌、误操作 | 人话提示 + 左右对比 + 分步完成 | MVP 先支持“按文件保留左/右”，手动编辑后续增强 |
| 恢复操作导致数据担忧 | 用户不敢点恢复 | 恢复前自动安全快照 + 清晰说明 | 若快照失败则阻止恢复并提示修复 |
| Git 输出异常导致解析失败 | 功能不可用 | Rust 层统一解析与错误码映射 | 输出原始诊断到日志，UI 给重试与导出日志 |
| 云端同步接入复杂 | 进度延期 | 同步入口先做结构占位 | MVP 仅本地版本保护，云端标记 Beta |
| 多语言与文案一致性 | 术语回退成黑话 | 术语映射表作为文案约束源 | 发布前文案审查清单 |

## 3. 验收映射（对齐用户目标）
- 保存进度：`当前修改 -> 保存进度` 主流程单屏可完成
- 查看历史：`保存记录` 页面可浏览与预览详情
- 切换方案：`我的方案` 可创建并切换
- 恢复版本：`保存记录` 可恢复并有安全保障
