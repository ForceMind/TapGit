# MVP 测试检查清单（Windows）

## 1. 环境检查
- [ ] 已安装 Node.js 22+
- [ ] 已安装 Git for Windows，`git --version` 可用
- [ ] `npm install` 成功，无阻断错误

## 2. 自动化检查
- [x] 单元测试可运行：`npm run test`
- [x] 构建可运行：`npm run build`
- [x] 目录打包可运行：`npm run pack:dir`
- [x] 安装包打包可运行：`npm run pack:win`

## 3. 核心流程手测
### 3.1 打开项目与开启版本保护
- [ ] 可选择本地文件夹
- [ ] 未开启版本保护时出现“开启版本保护”引导
- [ ] 点击后可正常进入后续功能

### 3.2 当前修改与保存进度
- [ ] 修改文件后可在“当前修改”页看到变更
- [ ] 可查看单文件变化详情
- [ ] 可执行“保存全部修改”
- [ ] 可执行“保存选中文件”

### 3.3 保存记录与恢复
- [ ] 保存后可在“保存记录”看到新条目
- [ ] 可查看记录详情（说明、时间、文件数）
- [ ] 可执行“恢复到这里”
- [ ] 恢复前有风险确认说明

### 3.4 新方案与合并
- [ ] 可创建新方案并自动切换
- [ ] 可手动切换已有方案
- [ ] 可把来源方案合并到目标方案
- [ ] 两边改到同一部分时可进入决策界面并完成处理

### 3.5 设置与日志
- [ ] 设置页可检测 Git 环境
- [ ] 可开关“首页新手引导卡片”
- [ ] 可切换安全快照开关
- [ ] 可导出日志文件

### 3.6 云端同步（基础版）
- [ ] 连接向导可生成标准 GitHub / GitLab 地址
- [ ] “测试连接”可返回通过/需授权/地址错误等提示
- [ ] 可填写云端地址并完成连接
- [ ] 可看到“本地待上传/云端待获取”数量
- [ ] 可执行“上传到云端”
- [ ] 可执行“获取最新内容”
- [ ] 未连接云端时有明确引导提示

### 3.7 语言与本地化
- [ ] 默认英文界面（非中文系统）
- [ ] 系统语言为中文时默认显示中文
- [ ] 在设置页切换语言后即时生效
- [ ] 中文模式下导航、顶栏、主要页面文案均为中文

## 4. 文案与可用性检查
- [ ] 新手主流程中不暴露 commit/push/branch 等术语
- [ ] 危险操作文案包含影响与恢复路径
- [ ] 空状态页面给出明确下一步按钮
- [ ] 错误提示包含可执行动作（重试/导出日志）

## 5. 打包产物验证
- [x] 生成 `release/TapGit-1.0.0.exe`
- [x] 生成 `release/win-unpacked/`
- [x] `win-unpacked/TapGit.exe` 可启动
- [x] 安装包启动后主进程无 `require("fs")` 相关启动错误
