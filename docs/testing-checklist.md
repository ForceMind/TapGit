# MVP 测试清单

## 说明

这份清单用于验证 TapGit 现阶段的主流程是否可用。

- 产品定位是“让普通人保存代码进度、恢复历史、试新想法、做基础同步”。
- 这不是 Git 全功能测试单。
- 常见 Git 能力覆盖情况请同时查看 [git-capability-matrix.md](./git-capability-matrix.md)。

## 环境检查

- [ ] 已安装 Node.js 22+
- [ ] 已安装 Git，且 `git --version` 可用
- [ ] `npm install` 成功，无阻塞错误

## 自动化检查

- [x] `npm run test`
- [x] `npm run build`
- [x] `npm run pack:dir`
- [x] `npm run pack:win`
- [ ] `npm run pack:mac`

## 主流程手测

### 打开项目与开启版本保护

- [ ] 可以选择本地项目文件夹
- [ ] 非 Git 项目会出现“开启版本保护”
- [ ] 开启后可以正常进入后续功能

### 从 GitHub 获取项目

- [ ] 可以填写 GitHub 仓库地址
- [ ] 可以选择本地保存位置
- [ ] 获取成功后可以直接进入项目
- [ ] 同名且非空文件夹会被阻止覆盖

### 当前修改与保存进度

- [ ] 修改文件后，能在“修改”页看到变更列表
- [ ] 能查看单个文件的差异详情
- [ ] 能保存全部修改
- [ ] 能只保存选中的文件
- [ ] 没有修改时不会误保存

### 历史与恢复

- [ ] 保存后，能在“历史”页看到新记录
- [ ] 能查看某条保存记录的详情
- [ ] 能恢复到某个历史点
- [ ] 恢复前有清晰的风险确认
- [ ] 恢复前会自动创建安全备份
- [ ] 能从安全备份回到恢复前状态

### 试新想法与合并

- [ ] 能创建新的试验副本
- [ ] 能切换到已有试验副本
- [ ] 能把试验副本带回稳定版本
- [ ] 遇到冲突时能进入决策界面
- [ ] 冲突界面支持逐个文件决定保留哪边
- [ ] 冲突界面支持批量保留稳定版本或试验副本
- [ ] 解决完成后能完成合并保存

### 云端同步

- [ ] 能查看 GitHub 登录状态
- [ ] 能在应用内发起 GitHub 登录
- [ ] 能在应用内退出 GitHub 账号
- [ ] 能测试云端地址是否可连通
- [ ] 连接测试能区分地址错误、权限不足、网络异常
- [ ] 能完成连接云端
- [ ] 能上传到云端
- [ ] 能获取云端最新内容
- [ ] 本地有未保存修改时会阻止直接获取最新内容

### 本地化

- [ ] 默认语言为英文
- [ ] 系统语言是中文时默认显示中文
- [ ] 设置中切换语言后立即生效
- [ ] 中文模式覆盖导航、顶栏、主要页面和确认弹层

## 发布链路检查

- [ ] Push 到 `main` 后自动生成 Windows / macOS 构建产物
- [ ] Push `v*` tag 后自动创建 GitHub Release
- [ ] Release 附带 Windows 与 macOS 安装包

## 当前明确不在 MVP 范围内

- [ ] 贮藏修改（stash）
- [ ] 重置最近一次提交
- [ ] 撤销某次提交（revert）
- [ ] 整理历史（rebase）
- [ ] 拣选提交（cherry-pick）
- [ ] 标签管理（tag）
- [ ] 强制上传（force push）
