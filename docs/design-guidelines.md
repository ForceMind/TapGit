# 码迹（TapGit）设计规范（MVP）

## 1. 视觉方向
- 风格定位：消费级桌面产品，温和、清晰、可恢复感强
- 关键词：`安全感`、`进度感`、`低门槛`
- 禁止：工程化图谱、过密信息墙、术语堆叠

## 2. 色彩系统
- 主色：`#0D6E6E`（稳重青绿）
- 主色浅层：`#E6F4F4`
- 强调色：`#FF8A3D`（提示进度和行动）
- 危险色：`#C23934`
- 成功色：`#1F7A1F`
- 文本主色：`#1F2328`
- 背景：`#F7FAFC` 到 `#EEF3F8` 渐变

## 3. 字体与排版
- 中文优先字体：`"Microsoft YaHei UI", "Segoe UI Variable", sans-serif`
- 信息层级：
  - 页面标题：24/30，600
  - 区块标题：18/24，600
  - 正文：14/22，400
  - 辅助信息：12/18，400
- 一屏优先原则：核心操作按钮应在首屏可见

## 4. 组件规范
- 卡片：圆角 12，轻阴影，悬停提升 2px
- 主按钮：高度 40，圆角 10，文案动词开头（如“保存进度”）
- 危险按钮：红色描边 + 二次确认弹窗
- 列表项：默认紧凑，选中态有明显背景差
- 差异块：删除为柔红背景，新增为柔绿背景

## 5. 动效规范
- 页面进入：200ms 淡入 + 上移 6px
- 列表加载：骨架屏闪动 800ms
- 成功反馈：右下角轻提示 2.5 秒自动消失
- 禁止夸张动画，避免增加认知负担

## 6. 可用性与无障碍
- 所有关键按钮必须有文字，不仅靠图标
- 颜色不是唯一状态表达，需配图标或标签
- 危险操作弹窗必须说明影响范围和恢复方式
- 键盘可达：Tab 顺序按视觉流

## 7. 新手模式 vs 高级模式
- 新手模式（默认）：隐藏 Git 技术细节，仅展示任务语言
- 高级模式（可选）：在详情区显示技术字段，不改变主按钮命名
- 模式切换不改变用户已保存数据
