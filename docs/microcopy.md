# 码迹（TapGit）文案规范（新手模式）

## 1. 文案语气
- 清晰、温和、可执行
- 先说“现在发生了什么”，再说“下一步做什么”
- 不训斥、不炫技、不抛工程术语

## 2. 核心按钮文案
- `开启版本保护`
- `保存进度`
- `保存全部修改`
- `保存选中文件`
- `恢复到这里`
- `创建新方案`
- `切换到这个方案`
- `合并到主线`
- `连接云端`
- `测试连接`
- `上传到云端`
- `获取最新内容`

## 3. 占位与提示文案
- 保存说明输入框：`这次主要改了什么？例如：修复登录按钮无响应`
- 空修改列表：`当前没有未保存修改。你可以继续编辑，改动会自动出现在这里。`
- 空历史：`还没有保存记录。完成一次“保存进度”后，这里会出现时间线。`
- 空方案：`目前只有主线方案。你可以创建一个新方案来试新想法。`

## 4. 错误与恢复文案
- 保存失败：
`这次保存没有成功。别担心，你的修改还在。`
`请重试，或先导出日志再反馈问题。`

- 恢复失败：
`恢复暂时没有完成，项目保持在恢复前状态。`
`你可以重试，或先创建一个新方案后再恢复。`

- 合并冲突：
`两边改到了同一部分，需要你决定保留哪一份内容。`
`你可以逐个文件处理，处理完成后再完成合并。`

## 5. 危险确认文案
- 恢复确认标题：`确认恢复到这个时间点？`
- 恢复确认正文：
`当前未保存修改可能被覆盖。`
`我们会先自动创建安全快照，方便你回到现在。`

- 合并确认标题：`确认把该方案合并到主线？`
- 合并确认正文：
`如果两边改到同一部分，我们会引导你逐项决定保留内容。`

## 6. 状态文案
- 成功：`已保存进度`
- 成功：`已切换到方案：{name}`
- 成功：`已恢复到：{time}`
- 成功：`合并已完成`
- 成功：`已连接云端地址`
- 成功：`连接测试通过，可以继续连接云端`
- 成功：`已上传到云端`
- 成功：`已获取云端最新内容`
- 提示：`本地有 {count} 条记录未上传`
- 提示：`云端有 {count} 条记录待获取`
- 提示：`云端地址可访问，但当前凭证不可用`
- 提示：`你还有未保存修改，请先完成“保存进度”再获取云端最新内容`
- 进行中：`正在读取修改内容…`
- 进行中：`正在保存进度…`
- 进行中：`正在合并方案…`
