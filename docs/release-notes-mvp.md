# 码迹 TapGit MVP 发布说明

发布日期：2026-04-09

## 1. 版本目标
本版本聚焦普通用户的四个核心动作：
- 保存进度
- 查看历史
- 切换方案
- 恢复版本

## 2. 新增能力
- 项目打开与版本保护启用
- 当前修改可视化与差异查看
- 保存进度（全部/部分文件）
- 保存记录时间线与详情
- 历史恢复（自动安全快照）
- 新方案创建、切换、合并
- 两边修改碰到一起时的可视化决策界面
- 设置页与日志导出
- 云端同步基础版（连接向导、连接测试、上传、获取、状态显示）
- 首页新手引导卡（默认开启，可在设置中关闭）
- 云端同步失败分场景引导（地址错误/权限不足/未连接/未保存等）
- 中英文界面支持（默认英文；系统语言为中文时自动中文；可在设置中手动切换）
- 中文模式覆盖导航、顶栏与主要操作文案
- GitHub Actions Windows 打包流程
- `v*` Tag 自动创建 GitHub Release 并附带安装包与便携包

## 2.1 修复
- 修复安装包在部分环境启动时报错：
  - `Calling require for "fs" in an environment that doesn't expose the require function`
  - 原因是主进程模块格式不一致
  - 现已统一为安装包可稳定启动的主进程格式

## 3. 技术信息
- 桌面框架：Electron
- 前端：React + TypeScript + Vite
- 状态管理：Zustand
- 测试：Vitest + Testing Library
- Git 接入：主进程 `simple-git` 封装

## 4. 验证结论
- `npm run test`：通过
- `npm run build`：通过
- `npm run pack:dir`：通过
- `npm run pack:win`：通过
- `release/win-unpacked/TapGit.exe`：可启动

## 5. 已知限制
- 云端同步当前为基础模式，暂未提供平台登录向导与多账号管理
- 未提供自定义应用图标
- 目前仅覆盖本地仓库主流程，复杂 Git 场景未开放

## 6. 下一阶段建议
- 增加 GitHub / GitLab 授权登录与仓库浏览器
- 优化大仓库差异加载性能（分页 + 按需加载）
- 增加合并决策辅助（按块决策、智能建议）
- 增加新手引导流程和首次使用演示
